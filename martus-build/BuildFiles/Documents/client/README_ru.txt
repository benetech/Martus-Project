А. Изменения в версии 3.3 по сравнению с версией 3.2

Данный выпуск доступен на английском языке и других языках, 
указанных на странице загрузки программного обеспечения 
сайта http://www.martus.org.  
На других языках данная версия будет доступна позже - в форме 
языковых пакетов на странице документации 
(http://www.martus.org/downloads)

- Добавлена возможность придавать одновременно нескольким сводкам 
  статус чистовых с помощью команды «Правка ->Сделать сводки чистовыми».
- Добавлена функция, позволяющая назначать главную учётную запись 
  по отношению к группе сводок или к папке со сводками. Для этого 
  введена команда «Правка ->Изменить доступ для главных учётных записей».
  Эта команда недоступна, если не выделена хотя бы одна сводка и 
  не назначена хотя бы одна главная учётная запись. Martus отображает 
  ход процесса изменения доступа в строке состояния; при желании 
  пользователь может прервать процесс изменения. В черновых сводках 
  будет обновлена Информация о главных учётных записях. Для каждой 
  чистовой сводки, к которой относятся изменения, Martus автоматически 
  создаст новую версию сводки, также чистовую. 
- Добавлена возможность просмотра вложенных изображений нескольких типов 
  непосредственно в Martus во время создания, предварительного просмотра 
  и редактирования сводок, а также при просмотре сведений о сводке.
- Исправлен ряд мелких ошибок, пользовательский интерфейс стал более 
  чётким и понятным.

ЗАМЕЧАНИЕ:  
Введена дополнительная возможность ускорить загрузку Martus и навигацию при 
большом количестве сводок для учётной записи (доступна начиная с версии
Martus 3.1, однако в Руководствах пользователя не упоминалась). 
В конец командной строки ярлыка Martus на рабочем столе можно добавить 
параметр " --folders-unsorted". Тогда при первоначальной загрузке папок 
сводки не будут отсортированы (а именно их сортировка при большом 
количестве сводок может занять значительное время). 
Впоследствии сводки всегда можно будет отсортировать в желаемом порядке 
нажатием на соответствующий заголовок столбца в окне списка сводок, 
однако упомянутая опция командной строки ускорит загрузку Martus и создание 


Б. Изменения в версии 3.2 по сравнению с версией 3.1


- Новые возможности отображения сводок позволяют: 1) показывать
  или скрывать поля в сводках - например, скрывать длинные поля,
  занимающие на экране много места; 2) создавать в сводках 
  разделы, которые также можно отображать либо скрывать; 
  3) отображать таблицы в "расширенном" виде (в котором также 
  можно добавлять строки); 4) располагать поля в строке друг за
  другом; 5) уменьшать принятые по умолчанию размеры таблиц, 
  чтобы сэкономить место на экране.
- Добавлена возможность требовать обязательного заполнения 
  конкретных полей для сохранения сводки. 
- Введена возможность создания "контекстных" выпадающих списков, 
  значения в которых будут формироваться из данных, введенных 
  в то или иное поле таблицы в любом месте сводки. Обратите 
  внимание, что источник данных и выпадающий список, формируемый 
  из них, не могут находиться в одном и том же поле таблицы.
- Добавлены диалог, отображающий ход процесса поиска, и 
  возможность отмены поиска.
- Повышена скорость выполнения поисковых и других операций со 
  сводками, что особенно важно для пользователей с большим 
  количеством сводок. 
- Решено несколько проблем, в частности: 1) устранены отказы 
  при сохранении настроечных данных - напр., главных учетных 
  записей - для пользователей, задавших очень большое 
  количество пользовательских полей в сводках;  2) устранены 
  ошибки, возникавшие при перетаскивании мышью сводок в процессе 
  отправки на сервер или загрузки с сервера.
- Улучшено содержание сообщений для пользователей (напр., 
  сообщений об ошибках при настройке пользовательских полей)
- Исправлен ряд мелких ошибок, пользовательский интерфейс 
  стал более чётким  и понятным.



В. Изменения в версии 3.1 по сравнению с версией 3.0

- Добавлена функция, уведомляющая пользователей главных учетных 
  записей о появлении на сервере новых загружаемых сводок, 
  сделанных под учетными записями, подчиненными им. Чтобы 
  активировать эту функцию в Вашей учетной 
  записи Martus, выберите команду Параметры > Настройки и 
  пометьте опцию 
  "Автоматическая проверка новых сводок от подчиненных". 
  Примерно каждый час в строке состояния (левый нижний угол 
  экрана) будет появляться сообщение о том, что Martus 
  проверяет новые сводки от учетных записей, подчиненных 
  Вашей. При наличии новых сводок будет появляться другое 
  сообщение, и Вы сможете зайти в меню Сервер для запуска 
  окна загрузки сводок.  
- Добавлена возможность сортировки сводок в окне загрузки 
  щелчком по заголовку столбца. Сводки сортируются только в 
  порядке возрастания.
- Повысилось быстродействие, усовершенствована работа с 
  памятью при входе в систему и выходе из Martus 
  пользователей с большим количеством сводок. 
- Исправлены следующие ошибки, имевшиеся в версии 3.0: 
  1) отсутствие диапазонов дат в постраничных отчетах, 
  2) потеря настроек полей в конфиденциальном разделе 
  сводки при выходе из Martus,  
  3) проблемы с выбором файлов при работе в ОС Mac (напр., 
  при настройке главных учетных записей, вложении файлов 
  в сводки, восстановлении ключа, импорте шаблонов настроек) 
  4) неверное отображение арабских дат в отчетах между 
  счетчиками количества сводок и подробными сведениями. 
- Исправлен ряд мелких недочетов, внесены уточнения и 
  исправления в пользовательский интерфейс.


Г. Изменения в версии 3.0 по сравнению с версией 2.9

- Добавлена возможность создания отчетов, отображающих 
  результаты поиска сводок по конкретным критериям. Отчеты 
  можно распечатывать или сохранять в файл.  Отчеты могут 
  содержать подмножества полей сводок. Их можно форматировать 
  в виде таблицы, где одна строка соответствует одной сводке.  
  Отчеты можно группировать и сортировать по нескольким полям 
  с подсчетом количества сводок в каждой группе.
- Добавлена возможность настройки формата нижнего раздела 
  сводки ("Конфиденциально").
- Новые функциональные возможности импорта позволяют 
  пользователям импортировать электронные данные в формат 
  сводки Martus, включая как текст, так и вложенные файлы.  
  Функциональные возможности экспорта также обновлены, с 
  тем чтобы соответствовать структуре импортируемых данных, 
  и теперь позволяют экспортировать вложения.
- Расширенная функциональность поиска теперь включает 
  возможность поиска по конкретным столбцам созданной 
  пользователем таблицы, объединения схожих полей в списке 
  выбора поля для поиска, а также разъяснений в тех случаях, 
  когда различные поля имеют одинаковые метки.
- В полях дат в сводках значение года по умолчанию, ранее 
  соответствовавшее текущему году, теперь изменено на "неизвестно"
- Существенно повышена производительность работы для учетных 
  записей с большим количеством сводок, особенно при выводе 
  экрана загрузки сводок.    
  Введены дополнительные сообщения в строке состояния, 
  информирующие пользователей при выполнении потенциально 
  длительных операций.
- Усовершенствована настройка полей, в том числе введены 
  дополнительные сообщения для пользователей и многострочное 
  отображение длинных меток пользовательских полей


Д. Изменения в версии 2.9 по сравнению с версией 2.8.1

- В версии сервера 2.6 Server (март 2006 г.), повышена скорость 
  нескольких основных операций клиент/сервер. Ускорились 
  следующие операции: выгрузка/отправка сводок, загрузка 
  собственных сводок и сводок подчиненных учетных записей, 
  инициализация при подключении к серверу при входе в систему.
- Повысилась производительность работы с учетными записями 
  при большом числе сводок.  Это касается скорости выполнения
  следующих действий:  отображения папок, сортировки, 
  перемещения сводок и т.п.
- Операции загрузки с сервера изменены таким образом, что они 
  выполняются в фоновом режиме (аналогично отправке сводок на 
  сервер), так что можно продолжать текущую работу в Martus, 
  не дожидаясь окончания загрузки. По окончании загрузки сводки 
  отображаются в соответствующей папке - "Загруженные". Для 
  отмены загрузки следует вернуться в диалоговое окно загрузки.
- Обновлены сообщения о статусе сервера.
- Добавлена возможность поиска по столбцам в таблице (а не по 
  всему тексту таблицы) при выборе полей в режиме расширенного 
  поиска, а также возможность поиска только по последним 
  версиям сводок.
- Исправлены ошибки функции поиска, имевшиеся в версиях 2.8 
  и 2.8.1. Исправлен поиск по выпадающим спискам с элементами,
  содержащими пробелы. В списках полей для поиска, если метка 
  пользовательского поля пуста (например, для полей заголовков 
  разделов), используется тег этого поля, чтобы обозначение 
  поля в списке для поиска не оказывалось пустым. 
- Добавлена возможность вставки и удаления строк в 
  пользовательских таблицах и в экране поиска
- При отображении данных в сводках и диалога "Контактная 
  информация" используется все доступное экранное пространство
- Команда "Отправить сводки на сервер заново" перемещена в меню 
  "Сервер" (для главных учетных записей, сохраняющих на серверах 
  сводки своих подчиненных учетных записей, пользователи которых 
  не имеют самостоятельного доступа к Интернету)
- Упразднены сообщения "Не все сводки были загружены", которые 
  могли вводить пользователя в заблуждение. Они отображались в 
  случаях, когда главная учетная запись не имела права просмотра 
  старых версий конкретных сводок.
- Несколько изменений в настройках даты: локализация форматов 
  даты, дополнительные возможности выбора формата, а также 
  изменения, позволяющие правильно отобразить даты (и 
  преобразовать ранее введенные даты) на тайском и персидском 
  языках. При работе с персидским языком используется известный 
  арифметический алгоритм для расчета високосных годов. Также 
  создано средство для помощи при диагностике настроек даты.  
- Внесено изменение, позволяющее текстовым процессорам правильно
  отображать диакритические знаки в html-файлах отчетов.
- В список языков добавлен курдский.  Если у Вас имеются проблемы 
  при отображении курдских шрифтов в Martus, просьба сообщить об 
  этом по электронной почте help@martus.org .
- Реализована начальная версия средства импорта данных, 
  позволяющего преобразовывать электронные текстовые файлы 
  (формат .csv или .xml) в формат сводок Martus.  Данная начальная 
  версия не поддерживает импорт пользовательских таблиц и вложений, 
  однако будет поддерживать импорт всех остальных типов полей.  
  Чтобы получить инструкции и подсказки по применению этой функции, 
  отправьте запрос по электронной почте help@martus.org .
- Исправлены многочисленные мелкие ошибки, пользовательский 
  интерфейс стал более четким  и понятным.


Е. Изменения в версии 2.8.1 по сравнению с версией 2.8

В версии 2.8.1 могут возникнуть проблемы при поиске 
по пользовательским полям – выпадающим спискам, если элементы 
списков содержат пробелы. Martus корректно выполнит поиск 
сводок, если искать строки, содержащиеся в элементах списков, 
с использованием параметров «Любое поле» и «Содержит»; 
проблема возникает только, если в качестве области поиска 
заданы именно выпадающие списки. В следующем выпуске эта 
ошибка будет исправлена, пока же рекомендуется при создании 
пользовательских выпадающих списков не использовать пробелы 
в их элементах (при желании можно разделять слова в элементе 
списка знаками тире или подчеркивания вместо пробелов.)

- Исправлена имевшаяся в версии 2.8 неполадка, когда даты или 
  диапазоны дат, предшествующих 1 января 1970 г., отображались
  и могли сохраняться с ошибками
- Реализованы переводы пользовательского интерфейса версии 2.8
  на тайский и русский языки


Ж. Изменения в версии 2.8 по сравнению с версией 2.7.2 

Этот выпуск доступен на английском и других языках, перечисленных 
на странице загрузки программного обеспечения сайта 
http://www.martus.org.  На других языках продукт будет доступен 
позднее в виде языковых пакетов на странице документации (http://www.martus.org/downloads)

- Добавлена возможность создания в таблицах столбцов различных 
  типов (выпадающие списки, флажки, поля даты, поля диапазона дат)
- Расширенная функция поиска позволяет пользователям, помимо 
  поиска по всему тексту сводки) осуществлять поиск по конкретным 
  полям, в том числе пользовательским. Возможен поиск по нескольким 
  заданным полям (с помощью опций "и" и "или").
- Расширенные возможности печати, позволяющие одновременно 
  отправлять на печать несколько сводок 
- Добавлена возможность сохранения выбранной сводки (сводок) в 
  формате HTML, включая либо исключая конфиденциальные данные
- Добавлен новый пункт меню "Организация папок", позволяющий 
  пользователям располагать папки в желаемом порядке
- Добавлен вывод предупреждений в случаях, если запускаемый 
  пользователем перевод не соответствует версии программного 
  обеспечения. Также в окне "О программе" выводится дата выпуска 
  языкового пакета
- При запуске программы выводится логотип Martus, оповещая 
  пользователей, что программа загружается
- Обновлены библиотеки шифровки, используется Bouncy Castle 
  Java Cryptography Extension
- Дополнительно усовершенствован вывод на экран и на печать 
  информации на языках, в которых чтение выполняется справа 
  налево
- Встроены системные шрифты для показа меню на конкретных 
  языках (напр., на непальском)
- Свойства сводки изменены так, что при редактировании 
  черновика в него автоматически добавляются новые главные 
  учетные записи и пользовательские настройки
- В новом выпуске Martus Server 2.4 добавлена возможность для 
  пользователей клиентского программного обеспечения Martus 
  получать новости с сервера Martus, на который отправляются

Т. Изменения в версии 1.0.2 по сравнению с версией 1.0.1

- Когда вы выбираете изменение чистовой сводки, программа делает 
  копию сводки. В прежней версии, если исходная сводка содержала 
  вложения, процесс копирования мог привести к повреждению копии, а 
  иногда и исходной сводки. Эта ошибка теперь исправлена.
- При установке с компакт-диска возможно воспроизведение шаблона 
  для поля "Подробности". Если на компакт-диске имеется файл 
  DefaultDetails.txt, он будет скопирован в каталог Martus в 
  процессе установки программы. А затем, если при создании новой 
  учетной записи в каталоге Martus будет найден файл 
  DefaultDetails.txt, пользователь сможет использовать его 
  содержимое в качестве шаблона для поля "Подробности" во всех 
  создаваемых сводках.
- Код и внутренняя пакетная структура реорганизованы в целях 
  упрощения их поддержки.

Ф. Специальные инструкции: как обновить Winsock в случае проблем с 
   запуском программного обеспечения Martus под Windows 95.

  Для работы Java требуется библиотека Microsoft Winsock 2.0. Скорее 
  всего, эта или более поздняя версия Winsock уже имеется в Вашей 
  системе. Однако может оказаться, что на компьютере с Microsoft 
  Windows 95 используется более ранняя версия Winsock.
  Чтобы проверить, какая у Вас версия Winsock, найдите файл 
  "winsock.dll". 
  В меню "Файл" выберите "Свойства" и перейдите на вкладку "Версия".
  Если на Вашем компьютере с Microsoft Windows 95 установлена более 
  ранняя версия Winsock, чем 2.0, Вы можете воспользоваться 
  программой установки Winsock 2.0, расположенной в каталоге 
  Martus\Win95 .
  Если Вы интересуетесь этим вопросом, более подробную информацию о 
  том, как определить, установлены ли под Microsoft Windows 95 
  компоненты Winsock 2.0, можно найти по ссылке:

  http://support.microsoft.com/support/kb/articles/Q177/7/19.asp
